Sandbox Bridge Recovery
The bridge should auto-start after Windows sign-in on newly provisioned VMs
When to use this
You should not need this during normal use. `Bridge State` above is the main health check. Use this only if the VM was installed before autostart was configured, or if the watcher failed to launch after sign-in.
powershell -ExecutionPolicy Bypass -File "$env:USERPROFILE\Desktop\shared\bridge-tools\Start-X64dbgBridge.ps1"
Manual recovery command for the guest-side watcher that syncs x64dbg state, findings, and pending requests.
Captured Findings
How to read findings
Each finding is runtime evidence from the debugger flow. Summary tells you what happened, type tells you what category of event it was, and address tells you where to pivot next in decompilation or tracing.
Queued Requests
How to use queued requests
Requests are tasks for the sandbox debugger workflow. Action is the task name, address is optional targeting, notes explain intent, status shows where the task sits in the pipeline, and requested shows when it entered the queue.
Quick actions
How to queue debugger work
Use the chips for common tasks. Fill the form manually when you need a targeted request. Good novice defaults are: use action for the task name, add an address only if you know the exact location, and write notes that explain what evidence you want back.
